# 04 — 아날로그 회로와 신호 무결성 (Op-amp, PSRR, SI/PI)

> 목표: 0/1로 딱 떨어지지 않는 "아날로그" 세계. 실제 고속 칩에서 신호와 전원이 왜 지저분해지는지.

---

## 1. 왜 아날로그가 필요한가

디지털은 0/1이지만, **현실의 전기 신호는 연속적인 아날로그**다. 메모리에서도:
- 센스 앰프: DRAM 커패시터의 **수십 mV** 미세 신호를 증폭 (아날로그)
- 입출력(I/O): 고속으로 신호를 주고받을 때 파형 관리 (아날로그)
- 전원 회로: 안정적인 전압 공급 (아날로그)

즉 칩은 "디지털 로직 + 그걸 둘러싼 아날로그 회로"의 조합이다.

---

## 2. Op-amp (연산 증폭기) — 아날로그의 기본 블록

### 왜 알아야 하나
디지털 로직 자체엔 증폭기가 필요 없지만, **디지털 신호가 만들어지고 전달되는 물리적 경계**엔 반드시 아날로그 증폭 회로가 낀다:
- **DRAM 센스 앰프**: 커패시터의 수십 mV 미세 전하 차이를 0V/VDD로 증폭해야 "0인지 1인지" 읽을 수 있다 (단, 회로 구조는 아래에서 설명할 선형 증폭기가 아니라 **양(+) 피드백 래치**에 가깝다 — 4-4절 참고)
- **ADC**: 센서·마이크의 아날로그 신호를 디지털 회로가 다룰 크기로 증폭
- **PSRR·전원 노이즈 제거**: 아래 3절 전체가 사실 증폭기 안정성 문제
- **HBM 등 고속 I/O**: 배선을 지나며 약해진 신호를 받는 쪽에서 재증폭

Op-amp는 **두 입력의 차이를 매우 크게 증폭**하는 회로다. 단독으로 쓰기보다, 저항과 조합해 "원하는 배율의 증폭기"를 만든다.

![비반전 증폭기](images/opamp_noninv.png)

### 회로를 읽기 위한 최소 규칙
회로도는 "노드(node)의 전압을 알아내는 게임"이다. 규칙은 딱 이 정도만 알면 된다:
- **노드**: 순수 전선으로만 이어진 부분 = 항상 같은 전압. 중간에 저항 같은 소자가 끼면 그 소자를 경계로 서로 다른 노드가 된다.
- **옴의 법칙 (V=IR)**: 저항 하나에 걸리는 전압 = 흐르는 전류 × 저항값. 저항의 "정의"에 해당하는 식이다 (다른 법칙에서 유도된 게 아니라, 저항이란 부품이 이렇게 행동하도록 만들어진 것).
- **KCL(전류 보존)**: 한 노드로 들어온 전류 합 = 나간 전류 합. 전하가 사라지거나 생기지 않기 때문.
- **KVL(전압 루프)**: 한 바퀴 돌아 원래 자리로 오면 전압 변화 합 = 0.
- 이 두 법칙 때문에 **직렬 저항은 R이 더해지고(R1+R2), 병렬 저항은 역수가 더해진다(1/R1+1/R2)** — 직렬은 전류가 공유되는 구조라 전압이 나뉘고, 병렬은 전압이 공유되는 구조라 전류가 나뉜다.
- 저항 양 끝에 전압 차이가 있어도 전류가 흘러서 "같아지지" 않는다. 전원(또는 op-amp 출력)은 고정된 저장고가 아니라 **계속 채워주는 펌프**라서, 전류가 흐르는 만큼 계속 리필되어 전압 차이가 영구히 유지된다.

### 개방이득(Open-loop gain) — Op-amp 혼자만의 성질
삼각형 기호는 "이 부품은 `Vout = A × (V+ − V−)`를 만족한다"는 **정의**다 (저항 기호가 V=IR을 뜻하는 것과 같은 방식). 여기서:
- **A = 개방이득**: 피드백("루프") 없이 op-amp 혼자 가진 이득. 보통 10만 배 이상.
- A가 너무 커서, 피드백 없이는 입력의 아주 작은 차이도 출력을 전원 한계치(예: ±12V)로 확 붙여버린다(포화). 그래서 선형 증폭기로 쓰려면 반드시 **피드백**을 걸어야 한다.

### 피드백이 증폭기를 만드는 과정 — 가상 단락(Virtual Short)
Rf=9kΩ, Rg=1kΩ, Vin=1V, 전원 12V로 예를 들어보자.
1. Vin이 1V로 뛰면 + 입력=1V, − 입력은 아직 0V → 차이 1V × A(10만) = 포화 → **출력이 순간 +12V로 튐**
2. 출력 12V가 Rf·Rg 전압분배기(`V = Vout × Rg/(Rf+Rg)`)를 거쳐 − 입력에 되먹임: 12V × 1k/10k = **1.2V**
3. 이제 − 입력(1.2V)이 + 입력(1V)보다 커짐 → 차이가 음수로 뒤집혀 → 출력이 다시 낮아지는 방향으로 조정
4. 이 과정이 초고속으로 반복되며, **+ 입력과 − 입력이 정확히 같아지는 지점**에서 멈춘다: `1V = Vout × 1/10` → Vout = **10V**

이 "결과적으로 +와 −가 같아지는 현상"이 **가상 단락**이다. 두 입력이 전선으로 실제 연결된(진짜 단락) 게 아니라, 피드백 루프가 스스로 그 지점을 찾아가서 전압만 같아지는 것 — 그래서 "가상(virtual)"이다.

### 이득 공식이 나오는 이유
안정 상태에서 `V_(−) = V_(+)` 이므로:
```
Vin = Vout × Rg/(Rf+Rg)
→ Vout = Vin × (1 + Rf/Rg)
```
Rf, Rg는 설계자가 직접 고르는 값이라, **op-amp 자체의 부정확하고 거대한 A값과 무관하게 저항 비율만으로 정확한 배율을 설계**할 수 있다 — 이게 피드백의 핵심 이점이다.

- 비반전 증폭기 이득 = 1 + Rf/Rg (Vin을 + 입력에 연결)
- 반전 증폭기 이득 = −Rf/Rg (Vin을 − 입력에, 피드백 기준을 + 입력에 연결)

### +, − 기호의 의미
전압의 부호가 아니라 **핀 이름**이다. `Vout = A×(V+−V−)` 식대로, + 입력을 올리면 출력도 같이 오르고(**비반전**), − 입력을 올리면 출력은 반대로 내려간다(**반전**). op-amp 내부는 이미 이 두 핀이 고정된 부품이라 +와 +로는 못 만들지만, 외부에서 어떤 신호를 어느 핀에 연결하느냐로 비반전/반전 증폭기를 선택할 수 있다.

> 메모리에서 **센스 앰프**가 미세한 전하 차이를 확실한 0/1로 키운다는 "기능적 역할"은 이 증폭 원리와 같지만, 회로 구조 자체는 여기서 배운 선형(음성 피드백) 증폭기가 아니라 **양성 피드백 래치**에 가깝다 — cross-coupled 인버터 쌍이 작은 차이를 두 값(0/VDD) 중 하나로 급격히 밀어붙이는 정규화(regenerative) 비교기다.

💡 **시스템·SW 관점**
> 피드백 루프가 "출력을 계속 확인하고 조금씩 수정해서 균형점을 찾는" 과정은 **제어 시스템의 negative feedback loop**이자, ML로 치면 **gradient descent가 손실을 계속 확인하며 최적점에 수렴하는 것**과 같은 구조다. 다른 점은 속도뿐 — op-amp는 나노초 단위로 이 수렴을 물리적으로 해낸다.

## 3. PSRR — 전원 노이즈를 얼마나 버티나

### PSRR은 회로가 아니라 "스펙"이다
PSRR을 위한 별도 회로도는 없다. 이득(gain)·대역폭처럼, **이미 있는 회로(예: op-amp)가 전원 잡음을 얼마나 잘 막는지 재는 성능 지표(숫자)**다. 다만 원리를 이해하려면 2절에서 배운 회로에 숨은 세 번째 입력이 있다는 걸 봐야 한다: op-amp 내부 트랜지스터도 결국 VDD로 동작하므로, VDD가 흔들리면 그 흔들림이 V+, V−는 그대로인데도 Vout에 새어 들어간다.

```
전원 공급 핀: V+ (양전원), V− (음전원)   ← 회로도엔 보통 생략, 신호 입력 +/−와 이름만 같은 별개의 핀
```

> 실제 칩의 전원(VDD)은 깨끗하지 않다. 옆의 디지털 회로가 스위칭할 때마다 전원이 출렁인다. **PSRR(Power Supply Rejection Ratio)**은 이 전원 노이즈가 출력에 얼마나 새어 나오는지를 나타낸다.

### 신호 경로 vs 전원 경로 — 반대로 원하는 반응
| | 원하는 반응 | 관련 개념 |
|---|---|---|
| **Vin (신호, + 입력)** | 출력이 **크게** 바뀌어야 좋음 | 이득 = 1+Rf/Rg |
| **VDD (전원 핀)** | 출력이 **하나도 안** 바뀌어야 좋음 | PSRR |

같은 출력 핀(Vout)에서 섞여 나오지만, 어느 경로에서 왔느냐에 따라 "좋은 반응"의 정의가 정반대다. PSRR이 낮으면 이 둘이 구별 안 되게 섞여버린다.

```
PSRR(dB) = 20 · log10(전원 변동 / 출력 변동)
```
- PSRR **높을수록 좋다** (노이즈를 잘 막음). 보통 60~80dB = 노이즈의 1/1000~1/10000만 통과.
- dB는 로그 스케일(전압 비율은 20·log10 — 전력이 전압 제곱에 비례해서 계수가 2배). 예: PSRR=80dB → 비율 10,000배 → VDD가 100mV 출렁여도 출력엔 10µV만 샘.
- 고주파로 갈수록 PSRR이 나빠짐 → 그래서 **디커플링 커패시터**로 고주파 노이즈를 흡수.

> **메모리에서 왜 중요한가**: 센스 앰프가 수십 mV를 감지하는데, 전원 노이즈가 그만큼 새어들면 **읽기 오류**가 난다. PSRR이 낮으면 수율이 떨어진다. 신호가 작을수록(센스 앰프처럼) 같은 PSRR이라도 상대적으로 더 치명적이다.

### op-amp 전용 개념이 아니다
"전원을 받아 동작하면서 그 전원과 무관하게 정확한 출력을 내야 하는 모든 회로"에 적용되는 일반 개념이다: **전압 레귤레이터(LDO)**(뒷단에 깨끗한 전원을 공급하는 게 존재 이유라 PSRR이 핵심 스펙), **ADC/DAC**, **오실레이터/PLL**(전원 노이즈 → 클럭 지터), **전압 레퍼런스** 등.

## 4. SI/PI — 고속에서 신호와 전원이 지저분해진다

### 왜 이제 와서 "이상적인 선" 가정이 깨지나
지금까지는 "전선은 저항 0, 완벽하다"고 가정하고 노드 전압을 계산했다. 이 가정은 저속에서는 문제없지만, **칩·패키지·PCB에서 신호가 GHz로 빨라지면, 배선이 단순한 "선"이 아니라 저항·인덕턴스·커패시턴스를 가진 복잡한 전송선처럼 굴게 된다.** SI/PI는 바로 이 이상화가 무너지는 지점을 다루는 분야다.

### SI (Signal Integrity, 신호 무결성) — 신호가 깨지는 문제
- **반사(Reflection)**: 임피던스가 안 맞는 지점에서 신호가 되튄다 → 종단 저항으로 임피던스 매칭
- **크로스토크(Crosstalk)**: 옆 배선의 신호가 전자기적으로 새어 들어옴 → 간격 확보, 차동 신호 사용
- **ISI(부호간 간섭)**: 이전 비트의 잔상이 다음 비트를 방해 → 이퀄라이저

### PI (Power Integrity, 전원 무결성) — 전원이 흔들리는 문제
- **Ground/Power Bounce**: 여러 회로가 동시에 스위칭하면 순간 전류가 급변, 배선 인덕턴스 때문에 전압이 출렁 (V = L·dI/dt)
- **PDN(전원 분배망) 설계**: 전원 경로 전체 임피던스를 목표 이하로 유지
- **디커플링 커패시터**: 급한 전류 수요를 국소적으로 흡수 (SW의 캐시와 같은 역할)

#### V = L·dI/dt란
저항의 정의식이 `V=IR`(전압이 전류의 **크기**에 비례)이었다면, 인덕터(코일, 배선의 기생 성분)의 정의식은 전류가 **변하는 속도**에 비례한다:
- **L**: 인덕턴스(고유값). **dI/dt**: 전류가 시간에 따라 변하는 속도(순간변화율)
- 전류가 일정하면(dI/dt=0) 전압도 0. 전류를 급하게 바꾸려 할수록(dI/dt 큼) 전압 스파이크가 커짐

물의 관성 비유: 물이 일정한 속도로 흐르면 아무 힘이 안 필요하지만(관성은 있어도 유지엔 힘 불필요), **급하게 속도를 바꾸려면** 그 관성 때문에 확 저항하며 압력(전압)이 튄다. 실제 배선(본드와이어, PCB 트레이스)엔 작지만 0이 아닌 L이 있어서, 수백~수천 개 회로가 **동시에** 스위칭해 VDD 전류가 급변(dI/dt 급증)하면 이 작은 L도 무시 못 할 전압 스파이크를 만든다 — 이게 Ground/Power Bounce다.

### HBM에서 SI/PI가 특히 어려운 이유
1024개 통로가 동시에 스위칭 → 크로스토크·전원 노이즈가 극심. 게다가 적층 구조라 기생 성분이 일반 DRAM과 다르다. **차동 신호(differential pair)**로 노이즈를 상쇄한다.

💡 **시스템·SW 관점**
> - PSRR/SNR(신호 대 잡음비)은 **ML에서 배치 정규화가 없을 때 입력 분포 변화가 층을 거쳐 증폭되는 것**과 같은 "노이즈 누적" 문제 구조다.
> - 디커플링 커패시터 = **SW 캐시**(급격한 수요 스파이크를 로컬 버퍼로 흡수).
> - 물리 계층의 SI 문제와 SW 프로토콜 계층의 병목은 같은 방식(임피던스/용량 불일치)으로 추적할 수 있다.

## 메모리 칩에서의 적용
SI/PI는 회로설계 직무의 핵심 항목 중 하나다. HBM·고속 I/O 설계의 핵심 난제가 바로 여기서 나온다. Noise 성분 분석, Circuit Modeling이 이 영역에 속하며, 대규모 SI/PI 시뮬레이션 환경을 구축하고 AI로 가속하는 것(Multi-physics CAD)이 최근 트렌드다.
